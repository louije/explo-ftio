Critères de contrôle CriteresLegaliteOffreEmploi

Les trois champs textuels de l'offre soumis au contrôle de légalité : l'intitulé du poste, la description de l'offre et la présentation de l'entreprise. Chaque champ est analysé pour détecter des formulations problématiques (discrimination, mentions illégales, etc.).
intitule_offre (2–300 car.) · description_offre (max 3 900 car.) · description_entreprise (max 500 car.)
Réponse avec alertes LegaliteOffreEmploi · HTTP 200
Unique object · 1:1
Quand des problèmes sont détectés, l'API renvoie un code 200 avec la liste détaillée des alertes.
alerteDetailList[] — @ AlerteDetailList
Détail des alertes alerteDetailList[] · AlerteDetailList
1 ou plusieurs array · 1:N
Chaque alerte identifie le type de problème, le thème concerné, le nom de la règle violée, le détail de l'infraction, la phrase fautive et le champ de l'offre concerné.
typeAlerte · themeAlerte · nomAlerte · detailAlerte · phraseAlerte · champAlerte
Réponse sans alerte LegaliteOffreEmpIoi · HTTP 201
Unique object · 1:1
Quand aucun problème n'est détecté, l'API renvoie un code 201 avec un simple message de confirmation.
message — ex. "Aucune alerte identifiée"
Accès public — API accessible en client_credentials (royaume /partenaire). Scopes : api_jecontrolemonoffrev1 et legaliteoffreemploi. Aucun jeton usager requis. Les champs peuvent être envoyés en query string ou en body JSON.
Codes HTTP — 200 : alertes détectées (problèmes trouvés). 201 : aucune alerte (offre conforme). 400 : erreur fonctionnelle. 401 : non autorisé. 500 : erreur technique.

📝 Offre soumise

Secrétaire médicale — Lyon 3e

Le recruteur envoie trois champs : l'intitulé du poste, la description de l'offre et la présentation du cabinet médical. L'API analyse chaque champ indépendamment.

Les champs peuvent être envoyés en body JSON ou en query string.
⚠ 2 alertes détectées

Discrimination identifiée

DISCRIMINATION Critère d'âge → la phrase « entre 25 et 35 ans » dans le champ description_offre viole l'article L1132-1 du Code du travail
DISCRIMINATION Apparence physique → la mention « bonne présentation exigée » peut constituer une discrimination liée à l'apparence

Si aucun problème n'avait été détecté, l'API aurait renvoyé un HTTP 201 avec {"message": "Aucune alerte identifiée"}.
