Prérequis — Cette API est dans le royaume /agent (client credentials). Chaque appel nécessite le header ft-jeton-usager obtenu via l'API Rechercher usager, ainsi que les trois headers d'identification agent : pa-nom-agent, pa-prenom-agent, pa-identifiant-agent.
Flux de traitement — France Travail propose une sanction (PROP_SANCTION) → le Conseil départemental saisit la décision (SAISIE_SANCTION) → puis valide (VALID_SANCTION) ou refuse (REFUS_SANCTION). La suspension peut être levée après remobilisation (VALID_LEVEE_SUSP).
Concepts partagés — Voir les concepts croisés entre APIs dans le catalogue
Une douzaine de vocabulaires sensibles — Cette API formalise un cycle de vie de sanction qui implique plusieurs acteurs (France Travail, Conseil Départemental, CAF, manager FT, médiateur, tribunal administratif). Chaque événement du cycle porte son type, son motif, éventuellement sa conséquence (suspension / suppression / radiation), sa modalité (pourcentage ou montant), sa durée. Tous ces vocabulaires sont déclarés en enum dans l'OpenAPI.

Cycle de vie de la sanction 11 types d'événement · 2 vocabulaires de motif schémas Evenement · DetailEvenementSanction

Une sanction traverse plusieurs étapes : proposition → validation (ou refus) → éventuellement annulation ou levée de suspension. Chaque étape est un événement, avec son type et son motif. Deux vocabulaires de motif coexistent : un court (3 valeurs, simplifié) et un détaillé (18 valeurs, qui distingue qui a pris la décision : FT, CD, CAF, manager, médiateur, tribunal…).
Type d'événement11
  • PROP_SANCTION Proposition / initialisation de la sanction
  • SAISIE_SANCTION Choix d'instruire la proposition par le CD (suspension RSA uniquement)
  • REFUS_SAISIE_SANCTION Refus d'instruire par le CD (suspension RSA uniquement)
  • ACCORD_FT_SANCTION Accord du manager France Travail
  • VALID_SANCTION Décision de sanction
  • REFUS_SANCTION Sanction refusée
  • ANNULATION_SANCTION Annulation (recours, erreur, tribunal…)
  • PROP_LEVEE_SUSP Proposition de levée de suspension
  • ACCORD_FT_LEVEE_SUSP Accord levée de suspension par le manager FT
  • VALID_LEVEE_SUSP Décision de levée de suspension
  • REFUS_LEVEE_SUSP Refus de levée de suspension
typeEvenement — séquence type : PROP_SANCTION → [SAISIE_SANCTION | REFUS_SAISIE_SANCTION] → ACCORD_FT_SANCTION → [VALID_SANCTION | REFUS_SANCTION] → (éventuellement ANNULATION_SANCTION ou cycle levée suspension).
Motif — version courte3
  • DROIT_ERREUR Droit à l'erreur
  • RECOURS Annulation suite à recours
  • AUTRE Autre motif
motifEvenement (dans le schéma Evenement) — utilisé principalement pour ANNULATION_SANCTION.
Motif — version détaillée17
  • DROIT_ERREUR Droit à l'erreur (sanction prononcée CD ou CAF)
  • RECOURS Annulation suite à recours (sanction prononcée CD ou CAF)
  • AUTRE Autre motif (sanction prononcée CD ou CAF)
  • DELEGATION_FT Validation au titre de la délégation FT
  • DELAI_SAISIE_DEPASSE Délai d'instruction CD dépassé
  • REFUS_SAISIE Refus d'instruction par le CD
  • VALIDATION_CD Décision de sanction par le CD
  • REFUS_CD Proposition refusée par le CD
  • REFUS_ELD Proposition refusée par le manager France Travail
  • RECENTALISATION_RSA_OUTREMER Proposition à traiter CAF Outremer (recentralisation RSA)
  • VALIDATION_CAF Décision de sanction par la CAF
  • REFUS_CAF Proposition refusée par la CAF
  • ABANDON_RECOURS_FT Annulation suite réclamation FT (sanction prononcée FT)
  • MEDIATEUR Annulation suite préconisation médiateur (sanction prononcée FT)
  • TRIBUNAL_ADM Annulation suite décision tribunal administratif
  • ANNU_SANCTION Annulation suite erreur décision sanction (sanction prononcée FT)
  • ANNU_RECOURS Annulation suite erreur décision recours (sanction prononcée FT)
motif (dans le schéma DetailEvenementSanction) — vocabulaire riche qui porte à la fois quoi et qui (FT, CD, CAF, manager, médiateur, tribunal). Note orthographique : RECENTALISATION est l'orthographe du schéma (typo probable de « recentralisation »).

Codes de retour 7 codes de sortie · 28 codes d'erreur schémas Retour*

Réponses structurées du service. Les codes C0xx signalent un succès ou une incompatibilité métier ; les R0xx signalent des erreurs de validation (champs manquants ou invalides).
Codes de sortie (métier)7
  • C000 Demande de radiation créée (RetourDemandeRadiation)
  • C001 Événement publié / sanction partenaire créée (succès standard)
  • C002 (non documenté — RetourDemandeRadiation)
  • C003 (non documenté — RetourDemandeRadiation)
  • C004 (non documenté — RetourDemandeRadiation)
  • C999 Événement incompatible avec les événements connus pour cette conséquence
codeSortie — la signification varie selon le schéma (mise à jour événement, création de sanction partenaire, demande de radiation). Les libellés C002-C004 ne sont pas documentés dans l'OpenAPI.
Codes d'erreur de validation28

Série R0xx groupée par champ concerné. Pattern : numéro pair = « obligatoire », numéro impair = « invalide ».
