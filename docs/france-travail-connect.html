Prérequis — France Travail Connect est le fournisseur d'identité OAuth 2.0. L'usager se connecte à votre application via le bouton « Se connecter avec France Travail » (obligatoire par la licence). Après authentification, le token d'accès permet d'appeler /userinfo. Scopes : api_peconnect-individuv1 (obligatoire), openid (obligatoire), profile (nom/prénom/genre), email (adresse email).
Les champs sub et idIdentiteExterne sont identiques — ils représentent le même UUID stable. Ce doublon est un artefact historique de l'API. Le champ subname est systématiquement null et n'a pas de fonction connue.

Flux OAuth 2.0 — Cinématique d'authentification
1. L'usager clique sur le bouton « Se connecter avec France Travail »
Redirection vers authentification-candidat.francetravail.fr/connexion/oauth2/authorize?realm=/individu
2. Après connexion, France Travail redirige vers votre redirect_uri avec un code d'autorisation
3. Votre serveur échange le code contre un token d'accès
POST authentification-candidat.francetravail.fr/connexion/oauth2/access_token?realm=/individu
4. Avec le token, appelez GET /userinfo pour obtenir l'identité de l'usager
Connexion d'un demandeur d'emploi — Amélie Durand se connecte à une application partenaire avec son compte France Travail. Après avoir autorisé les scopes demandés, l'application récupère ses informations d'identité.
