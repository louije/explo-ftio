Prérequis — Cette API est dans le royaume /agent (client credentials). Chaque appel nécessite le header ft-jeton-usager obtenu via l'API Rechercher usager, ainsi que les trois headers d'identification agent : pa-nom-agent, pa-prenom-agent, pa-identifiant-agent. Le jeton est volatile (~1 journée).
Les noms de champs utilisent le préfixe m_ (convention héritée du SI historique). Le champ m_statut reflète la situation administrative courante — un usager « Radié » n'est plus inscrit comme demandeur d'emploi.
Concepts partagés — Voir les concepts croisés entre APIs dans le catalogue
Un enum et quatre nomenclatures de référence — Seul m_statut est déclaré en enum OpenAPI. Les quatre autres dimensions (motif d'inscription, catégorie, situation au regard de l'emploi, motif de clôture) sont des nomenclatures France Travail officielles portées par paires _code/_lib — mais leurs valeurs possibles ne sont pas documentées dans le schéma OpenAPI. Ce qui est listé ci-dessous provient uniquement des exemples de réponses ; les listes complètes sont définies ailleurs (référentiels métiers internes à France Travail).

Motif de clôtureliste partielle
  • 5 Refus d'action de reclassement
  • 25 Reprise d'emploi par l'agence sur emploi non précisé
  • MV Abandon action aide à la recherche d'une activité (suppression de 1 mois)
m_motif_cloture_code · m_motif_cloture_lib — valeurs extraites des exemples. Codes hétérogènes (numériques et alphabétiques). Renseigné uniquement si m_statut = Radié ou Cessé.
Limite documentaire — Les quatre nomenclatures partielles (motif_inscription, categ_inscription, situation_reg_emp, motif_cloture) renvoient à des référentiels internes France Travail. L'API retourne à la fois le code et son libellé (paires _code/_lib), donc un consommateur n'a pas strictement besoin du référentiel complet pour afficher la donnée — mais pour filtrer/analyser, il faut disposer du référentiel. Celui-ci n'est pas exposé par ce schéma.
