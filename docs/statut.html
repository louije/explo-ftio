Prérequis — Cette API est dans le royaume /individu (PE Connect). L’utilisateur doit s’être authentifié via France Travail Connect avec les scopes api_peconnect-statutv1 et statut. L’application accède aux données de l’utilisateur connecté — aucun header agent n’est nécessaire.
Cette API répond à une question binaire : l’utilisateur est-il inscrit comme demandeur d’emploi ? Le code 0 signifie non inscrit, le code 1 signifie inscrit. C’est l’une des APIs les plus simples du catalogue, souvent utilisée pour adapter le parcours utilisateur d’une application partenaire.

Deux cas de figure — L’API retourne un résultat binaire. Voici les deux réponses possibles : un utilisateur inscrit comme demandeur d’emploi, et un utilisateur qui ne l’est pas.
// GET /statut // Auth: Bearer token via France Travail Connect // Scopes: api_peconnect-statutv1, statut // ── Réponse — Demandeur d’emploi ── { "codeStatutIndividu": "1", "libelleStatutIndividu": "Demandeur d'emploi" }
// ── Réponse — Non demandeur d’emploi ── { "codeStatutIndividu": "0", "libelleStatutIndividu": "Non demandeur d'emploi" }
